Internal Reporting Automation with AI

Context: Within the team, we identified that generating campaign performance reports was highly manual and time-consuming.

Challenge: The goal was to reduce operational workload while delivering actionable insights faster to business stakeholders.

Solution:

  • Designed and implemented a Slack bot capable of understanding natural language queries.
  • The bot uses the OpenAI API to transform user questions into structured SQL queries.
  • These queries are then executed against a Druid database, retrieving live analytical data.
  • Results are automatically formatted and returned directly in Slack, making insights accessible in real time.

AI-Powered Legal Documentation Assistant

Context: The Legal Team maintained detailed documentation in Confluence, but accessing and understanding it quickly was difficult for employees. A more intuitive, conversational way to navigate this information was needed.

Challenge:

  • Make legal documentation more accessible and actionable in employees' daily workflow.
  • Ensure all answers are grounded in official Confluence sources.
  • Integrate the solution seamlessly into Slack, the company's main communication tool.

Solution:

  • Designed and implemented a Slack bot that answers questions about Legal Guides.
  • Integrated the OpenAI Assistants API to process natural language queries.
  • Retrieved and parsed Confluence pages using the REST API and BeautifulSoup, transforming them into clean text.
  • Built and connected an OpenAI vector store to enable semantic search over legal documents.
  • Linked the assistant with the vector store so that every answer is supported with source citations.
  • Implemented a citation system that includes clickable links to the original Confluence pages.
  • Logged all conversations into Google Cloud SQL (PostgreSQL) for traceability and further analysis.
  • Added robust error handling, retry logic, and detailed logging for production stability.

Key Learnings:

  • Gained practical experience integrating LLMs with private enterprise data via OpenAI vector stores.
  • Learned how to automate ingestion and verification of documents into a semantic search system.
  • Developed a workflow for automatic source citation to increase trustworthiness of responses.
  • Strengthened skills in cloud-based SQL persistence and logging of AI interactions.
  • Experienced building a bot through its entire lifecycle: ingestion → processing → semantic search → user interaction → storage.

AI Bot Hub – Centralized Slack Assistant Platform

Context: Different teams (e.g., Legal and AdOps) needed AI-powered Slack bots with very different purposes and logic. Running and maintaining each bot separately created complexity and duplicated effort.

Challenge:

  • Create a centralized framework to manage multiple AI Slack bots.
  • Ensure each bot retains its own domain-specific logic and vector stores (e.g., Legal Guides, AdOps workflows).
  • Provide a way to easily scale by adding new bots in the future without rewriting infrastructure.

Solution:

  • Developed an AI Bot Hub that unifies multiple Slack bots under a single orchestration layer.
  • Implemented separate modules for each bot:
    • Legal Bot: Answers legal-related questions using Confluence documentation and vector search.
    • AdOps Bot: Provides operational support and best practices for campaign management.
  • Each bot runs in its own threaded process, ensuring independent operation while sharing a common hub.
  • Configured bots with their own OpenAI Assistant IDs, vector store IDs, and Confluence pages, managed centrally in one settings.py.
  • Logging, error handling, and monitoring are centralized, simplifying operations.
  • Designed the hub to be extensible: new bots with new purposes can be added by registering them in the configuration file.

Key Learnings:

  • Learned how to design a modular architecture for AI assistants, balancing independence and centralization.
  • Practiced multi-threaded orchestration for long-running bots in Slack.
  • Gained experience in building a scalable AI hub that supports different business units while reducing maintenance overhead.
  • Strengthened skills in Slack API, OpenAI API, and Confluence integrations, applied across different use cases.

Slack-Integrated Performance Reporting Bot (Dify + BigQuery)

Challenge:

  • Developed a custom Dify plugin to securely query BigQuery, enabling the bot to process live data.
  • Implemented natural language processing so users could request reports without SQL knowledge.
  • Ensured results were formatted and actionable, delivered directly into Slack conversations.

Key Learnings:

  • Gained experience extending Dify with custom plugins for enterprise use cases.
  • Strengthened skills in BigQuery integration, secure API design, and Slack automation.
  • Learned how to balance usability for non-technical teams with backend complexity.
  • Delivered a solution that reduced manual reporting time and improved decision-making speed.
